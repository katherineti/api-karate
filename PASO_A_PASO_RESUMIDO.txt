  VALIDACIONES IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════════

Para Crear Solicitud:
✓ División debe existir
✓ División debe estar activa
✓ Categoría debe existir
✓ Categoría debe estar activa
✓ Alumno no puede tener solicitud duplicada

Para Subir Pago:
✓ Inscripción debe pertenecer al alumno
✓ Debe estar en estado 'pendiente' o 'en_espera'

Para Validar:
✓ Master debe ser creador del evento
✓ Inscripción debe existir

Para Rechazar:
✓ No puede rechazar si ya está pagado
✓ Master debe ser creador del evento

  ESTADOS Y TRANSICIONES

INSCRIPCIÓN (status):
┌─ PENDIENTE ─────────────→ VALIDADO ─────────→ ✅ CONFIRMADO
│    ↓                          ↑
│  RECHAZADO ◄────────────────┘
└─ Solo RECHAZADO si payment_status ≠ PAGADO

PAGO (payment_status):
┌─ NO_PAGADO ────→ EN_ESPERA ────→ PAGADO ─────→ ✅ COMPLETADO
│     ↓                ↓
│  Sin cambio     Espera validación
