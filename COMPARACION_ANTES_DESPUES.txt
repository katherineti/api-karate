ANTES (Incorrecto):
─────────────────────
✗ division_id (FK → event_divisions)       ← Innecesario, muy indirecto
✗ event_category_id (FK → event_categories) ← Poco explícito

─────────────────
✓ event_id (FK → events)                   ← Evento al que aplica
✓ category_id (FK → karate_categories)     ← Categoría del alumno
✓ modality_id (FK → modalities)            ← Modalidad: Kata, Kumite, etc.

ANTES (Complejo):
─────────────────
SELECT *
FROM tournament_registrations tr
LEFT JOIN event_divisions ed ON tr.division_id = ed.id
LEFT JOIN event_categories ec ON ed.event_category_id = ec.id
LEFT JOIN karate_categories kc ON ec.category_id = kc.id
LEFT JOIN modalities m ON ed.modality_id = m.id
WHERE tr.event_id = ?;

AHORA (Simple):
───────────────
SELECT *
FROM tournament_registrations tr
WHERE tr.event_id = ?;

│                       FLUJO DE INSCRIPCIÓN                                  │
└─────────────────────────────────────────────────────────────────────────────┘

ANTES (Confuso):
────────────────
1. Alumno solicita
   ├─ Envía: division_id
   └─ Sistema deduce: event_divisions → event_categories → karate_categories

2. Master formaliza
   ├─ Recibe: division_id, event_category_id (redundante)
   └─ Actualiza ambas columnas

AHORA (Claro):
──────────────
1. Alumno solicita
   ├─ Envía: event_id (SOLO el evento)
   └─ Sistema crea registro con category_id=NULL, modality_id=NULL

2. Master formaliza
   ├─ Recibe: category_id, modality_id (lo que realmente elige)
   └─ Actualiza ambas columnas directas
